# 🧠 Local Machine Learning Training Guide (Pipeline 2026)

This guide details the methodology, environment configuration, and execution pipeline required to train the predictive Machine Learning Engine (`MLBrain`) locally. Our system uses a specialized Random Forest architecture to classify directional market movements with **Multi-Timeframe awareness**.

## 📊 Dataset & Feature Engineering

The Predictive Engine depends on high-quality, normalized market data. We generate a dense matrix of **45 features** derived from multiple timeframes (1H, 4H, and 1D) emulated over 1h klines.

### Key Indicators (Native Bypass Optimization)
Due to environment compatibility, we use a native vectorized bypass for technical indicators:
*   **MACD**: Line, Histogram, and Signal across 1h, 4h, and 1d periods.
*   **Bollinger Bands**: Upper, Middle, Lower, and **Percentage Distance** (`feat_bb_pct_distance`).
*   **RSI**: Momentum force across 1h, 4h, and 1d.
*   **Order Flow**: Cumulative Volume Delta (CVD) and Sweep patterns (High/Low).
*   **External Context**: Bitcoin Dominance and Macro Risk scoring.

> [!TIP]
> All machine learning features are dynamically ingested by the `MLBrain` if they carry the `feat_` prefix. This allows for seamless addition of new technical signals without refactoring the model code.

## 💻 Training Pipeline & Feature Selection

The training process is automated via `scripts/train_model.py`, which implements a "The Purge" (Feature Selection) strategy.

### Execution Command:
```bash
python scripts/train_model.py --symbol BTCUSDT --epochs 200 --local
```

### Workflow Steps:
1.  **Ingestion**: Loads the `.parquet` dataset.
2.  **Walk-Forward Validation**: Uses `TimeSeriesSplit` (5-folds) to simulate real-world trading without data leakage.
3.  **Feature Importance Ranking**: Calculates the predictive power of each of the 45 features.
4.  **The Purge (Feature Selection)**: Automatically drops any feature with an importance score **< 0.01**.
5.  **Final Export**: Retrains on the optimal feature set and exports the model to `models/brain_rf_v1.pkl`.

## 🛡️ Validation Methodology

To ensure robust performance, we strictly follow Time-Series best practices:

*   **No Data Leakage**: We never use future information to predict past states.
*   **Labeling (Triple Barrier Method)**: We assign labels (-1, 0, 1) based on price hitting Take Profit (TP) or Stop Loss (SL) boundaries within a specific time `horizon`.
*   **Reliability Baseline**: Our current baseline accuracy is approximately **31.5%** on 3-class classification, which is statistically significant for high-frequency crypto markets.
